## 1. PyBullet

### Install
```bash
pip install pybullet
```

### Run the demo
```bash
python demo_pybullet.py
```

### What the demo does
- Connects to the physics server (GUI window or headless)
- Loads a flat ground plane and a **KUKA iiwa robot arm**
- Sends position commands to all 7 joints
- Steps the simulation 300 times (≈1.25 s simulated)
- Reads back joint angles, velocities, and end-effector position
- Drops a red ball and watches it fall

### Key concepts learned
```python
p.connect(p.GUI)                      # start physics server
p.setGravity(0, 0, -9.8)             # set world gravity
p.loadURDF("kuka_iiwa/model.urdf")   # load a robot
p.setJointMotorControl2(...)          # command a joint
p.stepSimulation()                    # advance physics by 1 step
p.getJointState(robot_id, joint_idx) # read joint position/velocity
p.getBasePositionAndOrientation(id)  # read object pose
```

## 2. MuJoCo

### Install
```bash
pip install mujoco
```

### Run the demo
```bash
python demo_mujoco.py
```

### What the demo does
- Defines a **2-link robot arm** (shoulder + elbow) in MJCF XML
- Loads the model entirely from a Python string — no files needed
- Commands the arm through a 5-pose motion sequence
- Reads sensor data: joint angles, velocities, tip (end-effector) position
- Performs a forward kinematics query

### Key concepts learned
```python
model = mujoco.MjModel.from_xml_string(XML)  # load model
data  = mujoco.MjData(model)                  # create state container
data.ctrl[0] = 1.0                            # set actuator command
mujoco.mj_step(model, data)                   # advance physics
data.sensordata[...]                          # read sensor output
mujoco.mj_kinematics(model, data)             # FK without dynamics
```

### Optional: open the 3-D viewer
Uncomment the viewer block at the bottom of the script to see the robot move in a 3-D window.

## 3. Gazebo + ROS 2

### Install (Ubuntu 22.04 recommended)
```bash
# Step 1: Install ROS 2 Humble
# https://docs.ros.org/en/humble/Installation.html

# Step 2: Install Gazebo + TurtleBot3 packages
sudo apt install ros-humble-ros-gz
sudo apt install ros-humble-turtlebot3*

# Step 3: Source ROS 2
source /opt/ros/humble/setup.bash
```

### Run the demo
```bash
# Terminal 1: Launch Gazebo with TurtleBot3
export TURTLEBOT3_MODEL=burger
ros2 launch turtlebot3_gazebo turtlebot3_world.launch.py

# Terminal 2: Run the controller
python3 demo_gazebo_ros2.py
```

### What the demo does
- Creates a ROS 2 **Node** that communicates with Gazebo over topics
- Subscribes to `/odom` (robot position) and `/scan` (LIDAR)
- Publishes velocity commands to `/cmd_vel`
- Drives the TurtleBot3 in a **1-metre square** pattern
- Includes basic **obstacle avoidance** (stops if LIDAR sees < 0.3 m)

### Key concepts learned
```python
rclpy.init()                          # start ROS 2
node = Node("my_node")                # create a node
pub  = node.create_publisher(Twist, "/cmd_vel", 10)   # publish
sub  = node.create_subscription(Odometry, "/odom", cb, 10)  # subscribe
rclpy.spin_once(node)                 # process one round of callbacks
rclpy.shutdown()                      # clean up
```

## Simulator Decision Tree

```
Do you need full robot system simulation
(navigation stack, multiple sensors, real ROS integration)?
    YES → Gazebo + ROS 2

Do you need very accurate contact physics
(manipulation, dexterous hands, humanoids)?
    YES → MuJoCo

Do you need something simple & fast to install
(RL experiments, quick prototyping)?
    YES → PyBullet
```

## Recommended Next Steps

1. **PyBullet**: Try `pybullet_envs` — pre-built RL environments (HalfCheetah, Ant, etc.)
2. **MuJoCo**: Install `dm-control` (DeepMind's RL library built on MuJoCo)
   ```bash
   pip install dm-control gymnasium[mujoco]
   ```
3. **Gazebo**: Try the Navigation2 stack for autonomous navigation
4. **All three**: The `gymnasium` library wraps all of them into a standard RL API
